# Roadmap

## Milestone: Custom Domains
Finalize custom domain routing with live validation and DNS guidance.

## Milestone: Detailed Feedback
Explore better feedback for Ansible workflows and evaluate whether agent-based execution should replace them.

## Milestone: Harden Site Creation
Verify custom domains, improve error states and feedback, and make site creation more reliable.

## Milestone: Site Operations
Add missing site features like magic login, index control, and maintenance mode, and manage WordPress updates for core, plugins, and themes.

## Milestone: Backups
Add offsite S3-Backups with restore flow and automation.

## Milestone: Security
Add complete `wpvulnerability.com` checks for servers, sites, and plugins.

## Milestone: Performance
Implement additional performance features like caching, CDN support, image optimization, and related tuning.

## Milestone: Observability
Use the agent to collect traffic, response times, errors, logs, 404 tracking, uptime monitoring, and other relevant data for the dashboard.

## Milestone: Staging
Create staging from production and push staging to live.

## Milestone: Settings
Add the missing settings needed across the app.

## Milestone: Command Palette
Add command palette actions for all major workflows in the app.

## Milestone: Dashboard
Implement a useful dashboard for the home route.

## Milestone: Help
Add a basic help feature for the command bar.

## Milestone: Authentication
Harden user authentication for production.

## Milestone: Deployment Validation
Test a real production deployment using only the README steps, exactly as a user would.

## Milestone: Migration / Import
Import existing WordPress sites via a WordPress plugin and bring them into the normal Pressluft lifecycle.

## Milestone: Local Workflow Support
Add CLI support for recreating staging environments and connecting them to live sites.

## Milestone: Git Support
Explore ways to deploy from Git.

## Milestone: Platform Expansion
Add support for more providers and more server profiles.
